# Implementation Plan: Automated Release Workflow

**Branch**: `030-release-versioning` | **Date**: 2026-06-16 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/030-release-versioning/spec.md`

## Summary

Introduce `release-it` + `@release-it/conventional-changelog` as the release toolchain,
a `/release` Claude skill for guided human-reviewed releases, a `release.yml` GitHub Actions
workflow that publishes Docker images to `walefish/klaro` on every `vX.Y.Z` tag push, and
update `docker-compose.yml` to reference the Hub image rather than building locally.

## Implementation Tasks (high-level)

Detailed tasks will be generated by `/speckit-tasks`. The logical sequence is:

1. **Install release-it deps** — add `release-it` and `@release-it/conventional-changelog`
   as workspace-root devDependencies
2. **Configure release-it** — create `.release-it.json` with conventional-changelog preset,
   branch guard, npm/github release disabled
3. **Update package.json** — add `"version": "1.0.0"` and `"release": "release-it"` script
4. **Create release.yml** — GitHub Actions workflow triggered by `v*.*.*` tags; login to
   Docker Hub; build and push `walefish/klaro:latest` + `walefish/klaro:${{ github.ref_name }}`
5. **Update docker-compose.yml** — replace `build: .` / `image: pcm` with
   `image: walefish/klaro:latest`
6. **Create /release skill** — `.claude/skills/release.md` implementing the guided release
   flow: branch check → dry-run preview → confirm → run release → verify → present output
7. **Seed initial CHANGELOG** — run `pnpm run release --dry-run` to validate, then document
   the first-release bootstrapping process in the skill
